利用GitHub搭建博客

时间：2016-2-25

介绍在window系统下用GitHub搭建个人博客，用较简单、较直接的方法搭建，比较合适会点前端的同学。

1、到https://github.com/上申请一个账号。
2、下载git安装（安装过程就是一阵回车，不多说了），可百度下载，或GIT官网下载。
3、先建一文件夹，假设为myGitHub。右键文件夹空白处，选择“Git Bash here”。
敲入以下代码，user.name为你github的账号，user.email为注册的邮箱。注意：账号有双引号，邮箱不用双引号！
git config --global user.name "你的账号"
git config --global user.email 邮箱
4、生成公钥（邮箱要有引号）。 
ssh-keygen -t rsa -C "邮箱"
找到上图中绿下划横的文件，图中目录为C:/Users/.../.ssh/id_rsa.pub，（.ssh文件夹有可能被隐藏，自行找出来）用文本编辑器打开id_rsa.pub文件，全选后复制。

5、登录GitHub，按下图步骤操作。 Title:起个名字（最好是英文的）；Key：把公钥粘贴进来。保存后如下图。 创建一个新仓库。 设置仓库，配置页面生成器（按钮）。 继续，换一个风格（默认就行，因为要自己做，所以选啥都一样）。 访问的网址已经生成，点击看看是不是有页面了？ 6、把gh-pages设为默认分支，以便git操作。 选择SSH,按“按钮”把地址复制下来。（到此GitHub上的设置已完毕）

7、回到myGitHub文件夹，初始化。 
git init
8、给刚才复制的SSH地址设个别称，以方便使用。（看大家都用origin，我也跟着...） 
git remote add origin SSH地址
9、把GitHub上的代码clone下来。 
git clone SSH地址
clone下来后，看到了文件夹和文件，我们进去改一下html文件（增加、删除都可以），随意发挥了！ 10、发布。记得把git的路径指到文件夹内，不然会提交失败。 git发布三连击！！！(其它git命令自己去查） 
git add .
git commit -m "修改信息"
git push origin gh-pages
11、打开你的网址，看看是不是已经成功了！哈哈，接下来就自由发挥啦！人有多大胆，地有多高产！！！
